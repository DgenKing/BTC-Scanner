# BTC Scanner — Design Document

## Project Overview
BTC Scanner is a confluence-based Bitcoin trading signal generator with sentiment analysis, paper trading, backtesting, and AI-powered strategy optimization.

## Architecture

```
btc-scanner.py          Main scanner — live signals, Telegram alerts, DeepSeek AI analysis
data_sources.py         Sentiment aggregator — Fear&Greed, Reddit, CoinGecko, Google Trends
paper_trader.py         Virtual portfolio — position management, TP/SL, trailing stops
backtester.py           Historical replay — runs signals against past data via Binance klines
optimizer.py            AI optimizer — MiniMax M2.5 generates & tests config variations
btc-scanner.conf        Live config (NEVER modified by optimizer)
btc-scanner_testN.conf  Test configs generated by optimizer
```

## Signal Pipeline

```
get_klines() → compute_indicators() → evaluate_trade_setup() → action (LONG/SHORT/WAIT)
```

### Indicators (6 confirmations)
1. **RSI** — Oversold (<35) for LONG, Overbought (>65) for SHORT
2. **MACD** — Bullish crossover for LONG, Bearish crossover for SHORT
3. **Support/Resistance** — Near support with MIN_SR_TOUCHES for LONG, near resistance for SHORT
4. **Fibonacci** — Price at fib retracement level (0.382, 0.5, 0.618, 0.786)
5. **Volume Profile (HVN)** — Price at High Volume Node
6. **Trend Filter (EMA)** — EMA50/EMA200 trend gate blocks counter-trend trades

### Trade Decision
- Need MIN_CONFIRMATIONS (default 3) out of 6 to trigger signal
- Trend filter blocks LONG in downtrend, SHORT in uptrend
- Cooldown: 6 candles after stop loss before re-entering same direction

## Backtesting

```bash
python3 backtester.py --days 120 --timeframe 1h --verbose
```

- Fetches historical klines from Binance (paginated)
- 300-candle sliding window through all data
- Uses PaperTrader internally for position management
- Sentiment stubbed to NEUTRAL (not available historically)
- All results appended to permanent backtest_log.txt

## MiniMax M2.5 Strategy Optimizer

### Purpose
AI-powered parameter optimization. MiniMax M2.5 analyzes backtest results, identifies weaknesses, and suggests improved config parameters. Each suggestion is written to a separate test config and backtested automatically.

### MiniMax M2.5 API
- **Base URL**: `https://api.minimax.io/v1`
- **Endpoint**: `/chat/completions` (OpenAI-compatible)
- **Model**: `MiniMax-M2.5`
- **Reasoning**: `reasoning_split: true`
- **Key**: Hardcoded in btc-scanner.conf `[MINIMAX]` section

### New File: `optimizer.py`

#### Flow
```
1. Load btc-scanner.conf (baseline) + backtest_log.txt (recent results)
2. Call MiniMax M2.5 → "analyze results, suggest better parameters"
3. Parse JSON response → new config values + reasoning
4. Write btc-scanner_test{N}.conf (baseline + MiniMax's changes)
5. Run backtest against test config (120d, 1h)
6. Compare to baseline → log to optimizer_log.txt
7. Feed results back to MiniMax for next iteration
8. After all iterations → print best config found
```

#### API Call
```python
MINIMAX_API_KEY = "<key from config>"
MINIMAX_BASE_URL = "https://api.minimax.io/v1"
MINIMAX_MODEL = "MiniMax-M2.5"

def call_minimax(system_prompt, user_prompt):
    resp = requests.post(
        f"{MINIMAX_BASE_URL}/chat/completions",
        headers={
            "Authorization": f"Bearer {MINIMAX_API_KEY}",
            "Content-Type": "application/json"
        },
        json={
            "model": MINIMAX_MODEL,
            "messages": [
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt}
            ],
            "temperature": 0.7,
            "reasoning_split": True
        },
        timeout=120
    )
    return resp.json()["choices"][0]["message"]["content"]
```

#### System Prompt
Tells MiniMax:
- It is a BTC trading strategy optimizer
- Lists all tunable parameters with valid ranges
- Explains the confluence system (RSI + MACD + S/R + Fib + HVN + Trend)
- Explains TP/SL and how they affect win rate vs reward ratio
- Must respond with a JSON block of new parameter values + reasoning
- Goal: maximize profit factor and total P&L while keeping max drawdown < 5%

#### Config File Generation
Reads baseline `btc-scanner.conf`, replaces only the params MiniMax suggested, writes to `btc-scanner_test{N}.conf`. API keys, paper trading, and sentiment settings are copied unchanged.

#### CLI Usage
```bash
# Run 3 optimization iterations on 120 days of 1h data
python3 optimizer.py --iterations 3 --days 120 --timeframe 1h

# Run 5 iterations with verbose backtest output
python3 optimizer.py --iterations 5 --days 120 --timeframe 1h --verbose
```
